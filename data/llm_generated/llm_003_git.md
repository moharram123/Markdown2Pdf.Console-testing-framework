# Introduction to Git

Git is a distributed version control system that allows developers to track changes in source code during software development. It provides powerful features for collaboration, version tracking, and branching.

# Common Git Commands

Here are some commonly used Git commands:

| Command           | Description                             |
|-------------------|-----------------------------------------|
| `git init`        | Initialize a new Git repository        |
| `git clone`       | Clone an existing repository            |
| `git commit`      | Save changes to the local repository    |
| `git push`        | Upload local repository changes to the remote repository |

## Key Features of Git

- Distributed architecture
- Branching and merging
- Staging area
- Lightweight and fast
- Strong community support

## Example Git Workflow

Here's a simple workflow to get started with Git:

```bash
# Initialize a new repository
git init

# Add a file to the staging area
git add README.md

# Commit the changes
git commit -m "Initial commit"

# Push the changes to the remote repository
git push origin main
```
